Create api/auth/login.php. This endpoint accepts POST only, reads JSON body, looks up user by email, verifies password with password_verify(), sets session on success.

```php
<?php
// api/auth/login.php
// POST { "email": "...", "password": "..." }
// Returns: { "success": true, "data": { id, name, email, role }, "error": null }
//       or: { "success": false, "data": null, "error": "Invalid credentials" }

require_once __DIR__ . '/../connect.php';

// Only allow POST
if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    http_response_code(405);
    echo json_encode(['success' => false, 'data' => null, 'error' => 'Method not allowed']);
    exit;
}

// Read JSON body
$body = json_decode(file_get_contents('php://input'), true);
$email    = trim($body['email']    ?? '');
$password = trim($body['password'] ?? '');

if ($email === '' || $password === '') {
    http_response_code(400);
    echo json_encode(['success' => false, 'data' => null, 'error' => 'Email and password are required']);
    exit;
}

// Look up user by email (prepared statement — prevents SQL injection)
$stmt = mysqli_prepare($conn, 'SELECT id, name, email, password, role FROM users WHERE email = ? LIMIT 1');
mysqli_stmt_bind_param($stmt, 's', $email);
mysqli_stmt_execute($stmt);
$result = mysqli_stmt_get_result($stmt);
$user   = mysqli_fetch_assoc($result);
mysqli_stmt_close($stmt);

// Verify password
if (!$user || !password_verify($password, $user['password'])) {
    http_response_code(401);
    echo json_encode(['success' => false, 'data' => null, 'error' => 'Invalid credentials']);
    exit;
}

// Set session keys
$_SESSION['user_id'] = $user['id'];
$_SESSION['role']    = $user['role'];
$_SESSION['name']    = $user['name'];

// Return user (never return the hashed password)
echo json_encode([
    'success' => true,
    'data'    => [
        'id'    => (int) $user['id'],
        'name'  => $user['name'],
        'email' => $user['email'],
        'role'  => $user['role'],
    ],
    'error' => null
]);
```

--- FILE: api/auth/me.php ---
Returns current session user or 401 if not logged in. Used by the SPA on every page load to restore auth state.

```php
<?php
// api/auth/me.php
// GET — returns the currently logged-in user from session.
// Returns: { "success": true,  "data": { id, name, email, role }, "error": null }
//       or: { "success": false, "data": null, "error": "Not authenticated" }  (HTTP 401)

require_once __DIR__ . '/../connect.php';

if (empty($_SESSION['user_id'])) {
    http_response_code(401);
    echo json_encode(['success' => false, 'data' => null, 'error' => 'Not authenticated']);
    exit;
}

// Re-fetch from DB so data is always fresh (role may have changed)
$stmt = mysqli_prepare($conn, 'SELECT id, name, email, role FROM users WHERE id = ? LIMIT 1');
mysqli_stmt_bind_param($stmt, 'i', $_SESSION['user_id']);
mysqli_stmt_execute($stmt);
$result = mysqli_stmt_get_result($stmt);
$user   = mysqli_fetch_assoc($result);
mysqli_stmt_close($stmt);

if (!$user) {
    // Session references a deleted user
    session_destroy();
    http_response_code(401);
    echo json_encode(['success' => false, 'data' => null, 'error' => 'Not authenticated']);
    exit;
}

echo json_encode([
    'success' => true,
    'data'    => [
        'id'    => (int) $user['id'],
        'name'  => $user['name'],
        'email' => $user['email'],
        'role'  => $user['role'],
    ],
    'error' => null
]);
```

--- FILE: api/auth/logout.php ---
Destroys the session. Frontend calls this, then redirects to #/login.

```php
<?php
// api/auth/logout.php
// POST — destroys the current session.
// Returns: { "success": true, "data": null, "error": null }

require_once __DIR__ . '/../connect.php';

if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    http_response_code(405);
    echo json_encode(['success' => false, 'data' => null, 'error' => 'Method not allowed']);
    exit;
}

// Clear session data and destroy the session cookie
$_SESSION = [];
if (ini_get('session.use_cookies')) {
    $params = session_get_cookie_params();
    setcookie(
        session_name(), '', time() - 42000,
        $params['path'], $params['domain'],
        $params['secure'], $params['httponly']
    );
}
session_destroy();

echo json_encode(['success' => true, 'data' => null, 'error' => null]);
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → api/auth/login.php | Untrusted email/password from client |
| $_SESSION → api/auth/me.php | Session ID from cookie — must verify user still exists in DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | login.php | mitigate | password_verify() against bcrypt hash; no timing attack since bcrypt is constant-time per hash |
| T-02-02 | Injection | login.php | mitigate | Parameterized query via mysqli_prepare — email never interpolated into SQL string |
| T-02-03 | Information Disclosure | login.php | mitigate | Hash never returned to client; "Invalid credentials" message does not distinguish wrong email vs wrong password |
| T-02-04 | Elevation of Privilege | me.php | mitigate | Re-fetches role from DB on every call — stale session cannot carry an old (higher) role |
| T-02-05 | Spoofing | logout.php | mitigate | Session cookie cleared with setcookie() so browser cannot replay old session ID |
</threat_model>
